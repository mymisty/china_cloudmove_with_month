## 总体判断

当前页面已经形成完整的课堂演示闭环：顶部主题信息、动态雨带地图、阶段讲解卡、播放控制、六阶段时间轴都有明确分工；Framer Motion 用于雨带位置、热点、时间轴进度和卡片切换，具备“动态地理教学演示”的基础高级感。

下一轮不应重做架构，建议围绕“讲台投屏时一眼看懂雨带怎么走”做强化：修正文案显示、增强雨带轨迹与方向感、把北进-停滞-北跳-南撤的节奏做成视觉层级，并让时间轴更像一条地理过程线而不是普通步骤条。

## P0：建议本轮 Builder 立即实现

### 1. 修复页面中文文案乱码

优先级：P0  
适合本轮实现：是  
原因：当前源码中的中文标题、标签、时间、说明、aria 文案均显示为乱码，若运行页面也复现，会直接破坏课堂可信度和可读性。

建议：
- 将 `App.jsx`、`RainBeltMap.jsx`、`StageCard.jsx`、`Controls.jsx`、`Timeline.jsx`、`rainBeltStages.js` 中所有乱码中文恢复为正确中文。
- 保持现有字段、组件结构和数据阶段不变，只做文本层修复。
- 修复后重点检查标题、阶段名、按钮 `title` / `aria-label`、地图区域标签、课堂讲解点、资料追踪等投屏可见文字。

验收标准：
- 首屏标题正确显示“中国雨带的季节移动”。
- 时间轴六个阶段、地图区域名、控制按钮辅助文本均无乱码。
- `npm run check` 与 `npm run build` 保持通过。

### 2. 强化雨带移动的“方向箭头”和“残影轨迹”

优先级：P0  
适合本轮实现：是  
原因：现有雨带本体有动画，但观众主要看到的是“当前雨带在哪里”，对“刚从哪里来、下一步往哪里去”的感知还可以更强。

建议：
- 在雨带主椭圆后方增加 1-2 层低透明度残影，表示上一阶段位置，尤其突出华南到江南、江淮到华北、华北到南撤的跳变。
- 在地图路径上增加当前阶段方向箭头：北进阶段向北，停滞阶段使用短波动或环形符号，南撤阶段向南。
- 南撤阶段建议使用与北进不同的路径色或箭头样式，例如青绿转琥珀，避免学生误以为仍在北上。

验收标准：
- 不点击文字，只看地图，也能区分“北进”“停滞”“北跳”“南撤”。
- 雨带主视觉仍是焦点，残影不喧宾夺主。

### 3. 把时间轴改成“节奏型过程线”

优先级：P0  
适合本轮实现：是  
原因：现有时间轴是六个均等步骤，但雨带过程并不是均速移动；课堂重点是北进、梅雨停滞、快速北跳、南撤。

建议：
- 在时间轴阶段上增加节奏标签：北进、缓慢北推、停滞、北跳、南撤、退出。
- 第 3 阶段“江淮梅雨”使用更宽或更稳的视觉状态，表现停滞蓄积。
- 第 4 阶段“华北东北雨季”用更明显的跃迁动效或高亮，表现北跳增强。
- 第 5 阶段开始，进度线方向或颜色应明显进入“南撤/收束”状态。

验收标准：
- 播放时，学生能从时间轴感知过程节奏，而不是只看到 1-6 编号。
- 仍保持现有 Timeline 组件，不引入复杂新交互。

### 4. 增加课堂投屏的“当前重点句”

优先级：P1  
适合本轮实现：是  
原因：右侧 StageCard 内容充实，但投屏讲解时最需要一句足够大的“本阶段结论”，方便教师指读和学生抄记。

建议：
- 在 StageCard 顶部或地图右上信息框中增加一条短句，例如“雨带在江淮附近稳定停滞，形成梅雨”。
- 内容可从 `classroomCue` 或 `explanation` 中提炼为 `keyPoint` 字段，也可先在组件内根据阶段写死，后续再数据化。
- 字号应明显大于普通说明，控制在一行或两行内。

验收标准：
- 每个阶段切换后都有一句醒目的教学结论。
- 不挤占地图主体区域，不造成右侧卡片信息过载。

## P1：本轮可做，若排期紧可后续

### 5. 统一气候视觉色板，减少“泛青色”单调感

优先级：P1  
适合本轮实现：可选  
原因：当前整体以 cyan/sky/teal 为主，干净但略偏单一；雨带、控件、进度线、背景都在同一色系，层级区分还可更精致。

建议：
- 保留青蓝作为雨带主色。
- 用琥珀色表达暖湿气流与梅雨蓄积，用深绿表达地形/陆地，用石板灰表达底图边界。
- 南撤阶段可引入少量暖色或低饱和绿色，形成和北进阶段的方向差异。
- 控制按钮和资料卡片减少大面积高饱和渐变，让地图动效成为视觉主角。

验收标准：
- 页面不变成单一青蓝色调。
- 色彩区分服务地理含义，而不是装饰性堆叠。

### 6. 提升地图的“中国东部地理识别度”

优先级：P1  
适合本轮实现：可选  
原因：当前 SVG 是示意底图，能承载雨带，但中国东部轮廓、海岸线、长江/黄河/秦岭淮河等参照物还可更清晰。

建议：
- 保持示意图风格，不引入重型 GIS。
- 增加长江、黄河、南岭/秦岭-淮河等轻量参照线，并用极简标签辅助定位。
- 区域标签避免压在雨带正中心，可随阶段高亮但保持地图可读。
- 在华北、东北、江淮、江南、华南的热点之间增加更明确的空间间距。

验收标准：
- 学生能更快判断雨带位于华南、江南、江淮、华北东北还是南撤区域。
- 地图仍像教学示意，不变成信息拥挤的行政区地图。

### 7. 优化播放节奏与阶段时长

优先级：P1  
适合本轮实现：可选  
原因：当前自动播放统一 4200ms，过程稳定但节奏没有体现“停滞更久、北跳更快、南撤收束”的差异。

建议：
- 为阶段数据增加可选 `duration` 字段。
- 江淮梅雨停滞阶段时长略长，华北东北北跳阶段切换更快但动效更强，南撤阶段使用稍慢的回落动画。
- 播放按钮旁可显示当前阶段剩余或阶段进度，但不要增加复杂控制面板。

验收标准：
- 自动播放时能感到“北进-停滞-北跳-南撤”的节奏变化。
- 手动点击仍即时响应。

## 本轮 Builder 推荐任务包

推荐 Builder 本轮只接收以下小范围增量任务：
- 文案乱码修复。
- RainBeltMap 中增加方向箭头、上一阶段残影、南撤差异化样式。
- Timeline 中增加节奏标签与停滞/北跳视觉强调。
- StageCard 或地图信息框中增加每阶段一句重点结论。

不建议本轮 Builder 做的事项：
- 不重写 React 组件结构。
- 不引入 GIS 地图库。
- 不大改数据来源与阶段顺序。
- 不新增复杂设置面板。
